# Buttons
Use when requiring the user to take an action.
Different types of buttons create different user expectations about the button's action...
**Hover and Click these examples to see interactive states**

## Levels
Use different levels to create a _hierarchy_ of importance
 - **Primary -** Reserved for the most important or suggested action of a UI. There should only be one per UI.
 - **Default (Secondary) -** Normal and default emphasis. Should be used also when there is no default action to take.
 - **Tertiary -** Low emphasis. Used for less important actions. Can be styled with accent or subtle palette (see `Type`).
